模块加载与卸载

在Linux内核开发中，模块化设计允许开发者动态扩展内核功能而无需重新编译整个内核。本文将深入解析模块的加载与卸载、参数传递机制，以及如何编写高效的Makefile，帮助开发者快速上手内核模块开发。

一、模块加载与卸载#

  1. 初始化与卸载函数入口

module_init与module_exit接口定义在linux/module.h头文件中。

• module_init()：定义模块加载时的初始化函数，如资源分配、设备注册等。必须声明为static int __init类型，返回0表示成功，负数表示错误码（如-ENOMEM）;

static int __init my_module_init(void) { ... }
module_init(my_module_init);

• module_exit()：定义模块卸载时的清理函数，如释放内存、注销设备等。必须声明为static void __exit类型，无返回值；

static void __exit my_module_exit(void) { ... }
module_exit(my_module_exit);
  1. 许可证与元信息声明

• MODULE_LICENSE()：声明模块遵循的许可证（如"GPL"），未声明会导致内核警告或限制符号导出。
• MODULE_AUTHOR()/MODULE_DESCRIPTION()：声明模块作者、功能描述等元信息，增强代码可维护性。

MODULE_LICENSE("GPL");
MODULE_AUTHOR("ymm");
MODULE_DESCRIPTION("A sample Linux kernel module");

二、模块参数传递#

  1. 定义模块参数

• module_param()：声明可通过命令行传递的参数，支持基本类型（int、char等）和权限设置（如S_IRUGO表示只读）。

static int debug_enable = 0;
module_param(debug_enable, int, 0644);  // 参数名、类型、权限
  1. 参数描述与数组支持

• MODULE_PARM_DESC()：为参数添加描述信息，便于modinfo查看。 • module_param_array()：定义数组参数，需指定数组长度变量。

static int ports[5] = {80, 443};
static int port_count = 2;
module_param_array(ports, int, &port_count, 0644);
MODULE_PARM_DESC(ports, "Network ports to monitor");

三、调试与日志#

  1. printk与dmesg

• 使用printk或pr_xxx在内核日志中输出信息，支持不同日志级别。打印相关接口定义在linux/printk.h头文件中。常用日至级别如下

log_levelprintkpr_xxx
infoprintk(KERN_INFO “”)pr_info()
warnprintk(KERN_WARNING “”)pr_warn()
errprintk(KERN_ERR “”)pr_err()
alertprintk(KERN_ALERT “”)pr_alert()

• 通过dmesg命令查看实时内核日志，或过滤特定模块信息。根据不同的日志等级观察到不同的打印字体。

dmesg

四、Makefile编写规范#

内核模块的编译需依赖内核头文件，典型的Makefile格式如下：

obj-m += my_module.o    # 指定模块目标文件
all:
    make -C /lib/modules/$(shell uname -r)/build M=$(PWD) modules
clean:
    make -C /lib/modules/$(shell uname -r)/build M=$(PWD) clean

• 关键说明：
• obj-m：编译为动态加载模块（.ko文件）。
• -C指定内核源码目录，M=$(PWD)指定模块源码路径。
• 多文件模块需指定依赖：

obj-m += complex_module.o
complex_module-objs := file1.o file2.o

五、模块操作命令#

• 加载模块：

sudo insmod my_module.ko <param>  # 手动加载（需处理依赖）
sudo modprobe my_module    # 自动处理依赖

• 卸载模块：

sudo rmmod my_module
sudo modprobe -r my_module

• 查看已加载模块：

lsmod | grep my_module

六、注意事项#

  1. 权限与依赖：加载/卸载需root权限，modprobe优于insmod（自动处理依赖）。
  2. 版本兼容性：确保模块与内核版本匹配，避免符号冲突。
  3. 安全性：避免强制卸载（rmmod -f），可能导致系统不稳定。
