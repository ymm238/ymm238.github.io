常见误区#

位移运算符#

  左移、右移优先级是低于加减运算的，如以下代码为：

    int a = 1;
    int b = a << 1 + 1;
    printf("b = %d\n", b);

  结果 b=2 。

  右移位数不可大于等于类型的宽度

    int i = 100;
    i = i >> 32;
    printf("i = %d\n", i);

  此时会报“right shift count >= width of type”。

取值后递加或递减#

  因为++、--的优先级高于*，所以想实现取指针数据并进行递加或递减时要加()。

    void func (int *count)
    {
        ...
        // *count++表示count地址后移动4个字节后再取值
        // 正确用法如下。
        (*count)++;
        ...
    }

结构体成员变量访问#

 成员访问操作符->的优先级高于类型转换操作符 (type) 与&。Node 结构体定义如下，

typedef struct {
    Child child;
    ...
} Node;

将结构体强制类型转换后获取成员变量。

    void *token = ...;
    Child c = ((Node *)token)->child;    //  正确；
    Child c = (Node *)token->child；     //  错误，此处为获取children后将其强转为parent；

获取成员变量的地址。

    Node *node = ...;
    Child child = &node->child;
